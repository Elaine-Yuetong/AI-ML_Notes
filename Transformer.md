
## 🧠 Transformer 是什么？

**Transformer是一种深度学习模型**，它的核心能力是：**看懂"序列"（一串东西）之间的关系**。

最初发明它是用来翻译语言的（比如中译英），后来发现它特别厉害，能处理各种"一串东西"的数据，比如：
- 一句话（一串词）
- 一段音乐（一串音符）
- 股票价格（一串时间点）
- **论文里的公式（一串符号）**

---

## 🔑 Transformer 最核心的思想

想象你**读一句话**的时候是怎么理解的：

> "小明 吃了 一个 苹果"

你不会只单独看每个词，而是：
- "小明"是**谁**？
- "吃了"是**动作**
- "一个"是**数量**
- "苹果"是**什么东西被吃了**

**关键在于**：你把每个词和所有其他词**联系起来**，才能理解整句话的意思。

### Transformer 就是这么干的！

它用一个叫 **"自注意力机制"（Self-Attention）** 的技术，让序列里的**每个元素都"看看"所有其他元素**，然后决定：
- 谁跟我关系密切？（重要！）
- 谁跟我关系不大？（没那么重要！）

---

## 💡 举个栗子 🌰

看这句话：

> "那个 追 小狗 的 男孩 跑 得 很快"

当你读到"男孩"的时候，你会想到：
- "那个"（指的就是它）⭐ 关系密切
- "追小狗"（它干的）⭐ 关系密切
- "跑得快"（它跑的）⭐ 关系密切
- 其他词关系没那么大

**Transformer 就是学会了自动计算这种"关系亲密程度"**，给每对词打一个"注意力分数"。

## 🏗️ Transformer 长啥样？（简化版）

```
输入序列:  [词1] [词2] [词3] ... [词n]
              ↓      ↓      ↓
        ┌─────────────────────┐
        │   自注意力机制       │ ← 每个词都"看"所有其他词
        │  (算出谁跟谁关系近)  │
        └─────────────────────┘
              ↓      ↓      ↓
        ┌─────────────────────┐
        │   全连接网络         │ ← 提炼信息
        │  (综合理解)          │
        └─────────────────────┘
              ↓      ↓      ↓
        输出新向量  [新1] [新2] ... [新n]
```

可以堆叠**很多层**，一层层加深理解。

## 🔬 Transformer 的三大核心组件

### 1️⃣ 自注意力机制（Self-Attention）

**干的事**：序列里每个元素问所有其他元素："咱俩关系多近？"

**举个栗子**：
```
序列: [Close] [ - ] [Open]
- "Close" 看 " - " ：嗯，它是操作符，我跟着它
- "Close" 看 "Open"：哦，咱俩是兄弟，都是操作数
```

### 2️⃣ 多头注意力（Multi-Head Attention）

**干的事**：不止从一个角度"看关系"，而是从**多个角度**同时看。

**再举个栗子** 🌰：
> "他 在 银行 存 钱"

- **头1**：看语法关系（"存"是动词，"钱"是宾语）
- **头2**：看语义关系（"银行"和"钱"有关系）
- **头3**：看位置关系（"他"在开头）

多个角度加起来，理解更全面！

### 3️⃣ 位置编码（Positional Encoding）

**问题是**：Transformer 同时看所有元素，它**不知道谁在前谁在后**！

**解决方案**：给每个位置加一个"位置标记"。

```
位置:   1      2      3
词:   [Close] [ - ] [Open]
位置编码: [p1]   [p2]   [p3]
最终输入: [Close+p1] [ -+p2] [Open+p3]
```

这样模型就知道"谁在前面，谁在后面"了。

## 📊 Transformer vs RNN/LSTM（传统序列模型）

| 对比项 | RNN/LSTM（老方法） | Transformer（新方法） |
|--------|-------------------|---------------------|
| **怎么读序列** | 一个一个读（像读课文） | 一次性全看完（像看全景图） |
| **能看多远** | 太长会忘（记忆有限） | 无限远（全都能看到） |
| **计算速度** | 慢（不能并行） | 快（能同时处理） |
| **理解能力** | 一般 | 超强 |
| **代表应用** | 早期语音识别 | ChatGPT、谷歌翻译 |

## 🚀 Transformer 为什么这么厉害？

### 1. 能看"长距离依赖"
```
句子：我 昨天 在 公园 里 遇到 了 一个 小时候 的 朋友
```
"我"和"朋友"隔了老远，但Transformer能一下子看出它们有关系。RNN可能会"忘记"前面的"我"。

### 2. 能并行计算
- RNN：读完第1个才能读第2个（串行）
- Transformer：同时读所有词（并行），**快得多**！

### 3. 理解上下文的能力超强
这就是为什么ChatGPT这么能聊——它用的就是Transformer的变体。

## 📝 把公式当序列 vs 当树

### 当序列（Transformer看的）
```
公式: (Close - Open) / Open
序列: [ (, Close, -, Open, ), /, Open ]
```
问题：`(a+b)` 和 `(b+a)` 序列不同，但意思一样。Transformer会以为它们是两个东西。

### 当树（RGCN看的）
```
        /
       / \
      -  Open
     / \
  Close Open
```
问题：不管写成 `(Close-Open)/Open` 还是 `(Close-Open)* (1/Open)`，树形结构都不一样，但RGCN能抓住"计算关系"的本质。

## 🔬 第一部分：自注意力机制（Self-Attention）—— 数学层面

### 🎯 核心目标

自注意力的目的是：**让序列里的每个词，都能"看到"所有其他词，并决定谁更重要**。

### 📊 计算步骤（一步一步来）

假设我们有个句子：**"我 爱 你"**

**Step 1：把每个词转成向量（数字）**

每个词用一串数字表示（这叫**词嵌入**）：
```
"我" → [1, 0.5]
"爱" → [0.2, 0.8]
"你" → [0.9, 0.3]
```
（实际中向量维度可能很大，比如512维，但这里用2维方便理解）

---

**Step 2：每个词生成三个"分身"**

对每个词，生成三个不同的向量：
- **Q（Query，查询）**：这个词想问别人什么？
- **K（Key，键）**：这个词能回答别人什么问题？
- **V（Value，值）**：这个词本身携带什么信息？

怎么生成？用**权重矩阵**乘上原向量。

```
对于"我"：
Q_我 = W_Q × [1, 0.5] = [0.8, 0.3]
K_我 = W_K × [1, 0.5] = [0.6, 0.4]
V_我 = W_V × [1, 0.5] = [0.7, 0.2]
```
（实际中Q、K、V维度可能不同）

**形象理解**：
- Q：**"我是谁？我在找谁？"**（主动提问方）
- K：**"我能回答谁的问题？"**（被动回答方）
- V：**"我实际要说啥？"**（最终要传递的信息）

---

**Step 3：计算注意力分数（谁和谁最亲密）**

对于每个词，用它的**Q**去和所有词的**K**做**点积（相似度计算）**。

```
"我"（Q_我）问所有词：
- 和"我"（K_我）的亲密分：Q_我 · K_我 = 0.8×0.6 + 0.3×0.4 = 0.48+0.12 = 0.60
- 和"爱"（K_爱）的亲密分：Q_我 · K_爱 = 0.8×0.5 + 0.3×0.7 = 0.40+0.21 = 0.61
- 和"你"（K_你）的亲密分：Q_我 · K_你 = 0.8×0.1 + 0.3×0.9 = 0.08+0.27 = 0.35
```

**Step 4：缩放（除以√d）**

除以**向量维度的平方根**，防止数字太大。
```
d = 2（我们用的维度）
√d = 1.414

缩放后的分数：
- 对"我"：0.60 / 1.414 = 0.42
- 对"爱"：0.61 / 1.414 = 0.43
- 对"你"：0.35 / 1.414 = 0.25
```

---

**Step 5：用Softmax转成概率（归一化）**

让所有分数加起来等于1，变成"注意力权重"。

```
原始分数: [0.42, 0.43, 0.25]
Softmax后: [0.34, 0.36, 0.30]
```
解释：
- "我"有34%的注意力给"自己"
- "我"有36%的注意力给"爱"（最关注"爱"！）
- "我"有30%的注意力给"你"

---

**Step 6：加权求和（得到最终输出）**

用"注意力权重"去加权所有词的**V（值）**。

```
"我"的新表示 = 0.34×V_我 + 0.36×V_爱 + 0.30×V_你
                = 0.34×[0.7,0.2] + 0.36×[0.3,0.5] + 0.30×[0.8,0.1]
                = [0.238+0.108+0.24, 0.068+0.18+0.03]
                = [0.586, 0.278]
```

**结果**："我"这个词的新表示，**融合了"我""爱""你"三个词的信息**，但"爱"的贡献最大（0.36）！

---

### 🎯 为什么要这么做？

| 步骤 | 目的 |
|------|------|
| Q、K分离 | 让模型灵活学习"谁该关注谁" |
| 点积计算 | 衡量两个向量的**相似度** |
| 缩放+Softmax | 把相似度转成**概率权重**（让它们总和为1） |
| 加权V | 把**重要信息**提取出来，**不重要的忽略掉** |

## 🔬 第二部分：多头注意力（Multi-Head Attention）

### 🎯 为什么要"多头"？

**问题**：用**单一**注意力，模型可能**只从一个角度看问题**。

比如：
> "他 在 银行 存 钱"

- **头1**：可能关注"存"和"钱"（动作-宾语关系）
- **头2**：可能关注"银行"和"钱"（地点-物品关系）
- **头3**：可能关注"他"和"存"（主语-动作关系）

**一个头只能看一种关系**，多个头就能**从不同角度同时看**！

---

### 📊 怎么做的？

**Step 1：多头分开算**

比如有 **8个头**（论文里常用），每个头**独立**做自注意力：
```
头1：Q1,K1,V1 → 注意力1 → 输出1
头2：Q2,K2,V2 → 注意力2 → 输出2
...
头8：Q8,K8,V8 → 注意力8 → 输出8
```
每个头的**权重矩阵W_Q, W_K, W_V都不同**，所以每个头学到的东西不一样。

---

**Step 2：把所有头的结果拼起来**

```
输出 = [输出1, 输出2, 输出3, ..., 输出8]  （拼接在一起）
```

**Step 3：再过一个线性层压缩**

用一个新的权重矩阵把这个**大向量**压缩回原来的维度。

---

### 🎨 形象理解

想象你在**面试一个人**：

| 头 | 关注角度 | 问题 |
|----|---------|------|
| 头1 | 技术能力 | 会不会写代码？ |
| 头2 | 沟通能力 | 表达清晰吗？ |
| 头3 | 团队合作 | 能和人共事吗？ |
| 头4 | 经验背景 | 之前做过什么？ |

**多头注意力**就像同时有**8个面试官**，每个人从不同角度提问，最后综合所有人的意见，做出全面的判断！

---

### 💡 多头注意力的好处

| 好处 | 解释 |
|------|------|
| **捕获不同关系** | 有的头关注"语法"，有的关注"语义" |
| **增强表达能力** | 模型容量更大，能学更复杂的模式 |
| **鲁棒性更好** | 一个头学偏了，其他头还能补救 |

## 🚀 第三部分：为什么Transformer这么能吃数据？

### 1. 并行计算（最大优势！）

**RNN/LSTM（老方法）**：
```
输入: [词1] → [词2] → [词3] → ... → [词n]
      ↑       ↑       ↑             ↑
    必须按顺序，一个一个算！慢死了！
```

**Transformer（新方法）**：
```
输入: [词1] [词2] [词3] ... [词n]
      ↓      ↓      ↓        ↓
    同时算！同时算！同时算！同时算！
      ↓      ↓      ↓        ↓
    输出: [新1] [新2] [新3] ... [新n]
```

**结果**：Transformer训练速度快10倍以上！

---

### 2. 长距离依赖（看得远！）

**RNN的问题**：
```
句子: "我 昨天 在 公园 里 遇到 了 一个 小时候 的 朋友"
      ↑                                       ↑
    第1个词                                第11个词
```
RNN要从"我"一个一个传到"朋友"，中间隔了10步，**早忘了"我"是谁**！

**Transformer的优势**：
```
"朋友"直接看所有词！
"朋友" ← "我"（直接连接！）
```
**每个词都能直接看到所有其他词**，不管你隔多远！

---

### 3. 可扩展性（越大越强！）

Transformer的**参数可以无限堆**：

| 模型 | 参数量 | 训练数据 |
|------|--------|---------|
| BERT-Base | 1.1亿 | 16GB文本 |
| BERT-Large | 3.4亿 | 16GB文本 |
| GPT-3 | **1750亿** | **45TB文本** |

**规律**：参数量越大，数据越多，模型越聪明！

**为什么能堆这么大？**
- 并行计算 → 可以用很多GPU同时训练
- 注意力机制 → 不依赖顺序，容易并行

---

### 4. 迁移学习（站在巨人肩膀上）

**传统方法**：每次任务都要从零开始训练 ❌

**Transformer做法**：
```
Step 1：在海量数据上预训练（比如全互联网文本）
         ↓
    得到一个"超级聪明"的基座模型
         ↓
Step 2：在你的小任务上微调（只需要少量数据）
         ↓
    得到任务专用模型 ✓
```

这就是为什么ChatGPT能聊天、能翻译、能写代码——它在**海量数据上先学了一遍**！

---

### 5. 位置编码（让模型知道顺序）

**问题**：Transformer同时看所有词，它**不知道谁在前谁在后**！

**解决方案**：给每个位置加一个"位置标记"。

```
原始词:  "我"  "爱"  "你"
位置编码: [p1]  [p2]  [p3]
最终输入: [我+p1] [爱+p2] [你+p3]
```

这样模型就知道"我"在第1位，"爱"在第2位，"你"在第3位。

**位置编码怎么算？**
用正弦/余弦函数生成，每个位置都有一个**独一无二的编码**。

## 🤖 第四部分：BERT和GPT分别是啥？

### 背景

BERT和GPT都是**基于Transformer**的模型，但**训练方式不同**，导致**用途不同**。

---

### 📖 BERT（Bidirectional Encoder Representations from Transformers）

**训练方式**：**完形填空**

```
原始句子: 我 爱 吃 [MASK] 果
           ↓
BERT要猜: 苹 果  ✓
```

**关键特点**：
- **双向**：看上下文（左边和右边都看）
- **理解任务**：主要用来"理解"文本

**应用场景**：
- 情感分析（这个评论是好评还是差评？）
- 问答系统（这个问题在文章里能找到答案吗？）
- 文本分类（这是新闻还是小说？）

**形象理解**：BERT是**阅读理解高手**，给你一篇文章，它能答出任何问题。

---

### 💬 GPT（Generative Pre-trained Transformer）

**训练方式**：**接龙**

```
原始: 我 爱 吃
      ↓
GPT要猜: 苹 果
      ↓
继续: 我 爱 吃 苹 果
      ↓
再猜: ， 很 甜
      ↓
继续: 我 爱 吃 苹 果 ， 很 甜
```

**关键特点**：
- **单向**：只能看左边（预测下一个词）
- **生成任务**：主要用来"生成"文本

**应用场景**：
- 聊天机器人（ChatGPT）
- 写文章、写代码
- 翻译、摘要

**形象理解**：GPT是**讲故事高手**，给它一个开头，它能接出整个故事。

---

### ⚖️ BERT vs GPT 对比表

| 对比项 | BERT | GPT |
|--------|------|-----|
| **训练方式** | 完形填空 | 接龙 |
| **看的方向** | 双向（左右都看） | 单向（只看左边） |
| **擅长任务** | 理解类 | 生成类 |
| **代表模型** | BERT, RoBERTa, ALBERT | GPT-1, GPT-2, GPT-3, ChatGPT |
| **典型用法** | "这篇文章讲了什么？" | "帮我写一篇文章" |
| **参数量** | 3亿左右 | 1750亿（GPT-3） |

---

### 🎯 为什么GPT比BERT火？

1. **生成能力更惊艳**：BERT只能"回答"，GPT能"创作"
2. **零样本学习**：GPT可以直接"听懂"新任务，不需要重新训练
3. **规模效应**：GPT系列越做越大，效果越来越好

---

### 🌰 具体例子

**任务：判断电影评论是好评还是差评**

**BERT的做法**：
```
输入: "这部电影太棒了！"
      ↓
BERT输出: 好评 (95%)
```

**GPT的做法**：
```
输入: "这部电影太棒了！这是一篇"
      ↓
GPT继续生成: "精彩的影评，强烈推荐！"
      ↓
然后人工或程序判断：好评 ✓
```

### 🔗 在AlphaSAGE论文里

论文里提到的**AlphaAgent**方法用了**LLM（大语言模型）**，其实就是GPT类的模型。

AlphaAgent的做法：
1. 用GPT生成"Alpha想法"（比如"我觉得收盘价和开盘价之差可能有用"）
2. 再用GPT把这个想法转成数学公式
3. 评估这个公式好不好，反馈给GPT改进

但AlphaSAGE没有用LLM，而是用了**RGCN + GFlowNets**，因为在公式挖掘任务上，**结构理解 + 多样化生成**比"生成式语言模型"更适合。

## 🔬 第一部分：为什么缩放要除以 √d？（数学原理）

### 🎯 问题回顾

在自注意力里，我们计算了 Q 和 K 的点积后，**除以了 √d**（d 是向量维度）。

```
原始分数 = Q · K
缩放后分数 = (Q · K) / √d
```

**问题**：为什么要多此一举？

---

### 📊 先看现象

假设我们有两个向量 **Q** 和 **K**，每个维度都是随机数，均值为0，方差为1。

**当 d=2 时**：
```
Q = [0.5, -0.3]
K = [0.2, 0.7]
点积 = 0.5×0.2 + (-0.3)×0.7 = 0.10 - 0.21 = -0.11  （正常范围）
```

**当 d=512 时**（实际Transformer用的）：
```
Q = [0.1, -0.2, 0.3, ..., 0.4]   (512个数字)
K = [-0.3, 0.5, 0.1, ..., -0.2]  (512个数字)
点积 = 0.1×(-0.3) + (-0.2)×0.5 + 0.3×0.1 + ... + 0.4×(-0.2)
     ≈ 0 + 0 + 0 + ... + 0  （每个乘积正负抵消）
     但！方差会累积！
     点积的方差 ≈ d × 1 × 1 = 512
     点积的标准差 ≈ √512 ≈ 22.6
```

**问题**：当 d 很大时，点积的数值会**变得非常大**（正负22左右）！

---

### ⚠️ 为什么大数值是问题？

因为接下来我们要做 **Softmax**：

```
Softmax(x_i) = e^{x_i} / Σ e^{x_j}
```

**如果 x 很大（比如 22）**：
```
e^{22} ≈ 3.6 × 10^9  （36亿！）
e^{-22} ≈ 2.8 × 10^{-10}  （几乎为0）
```

**结果**：
- 最大的那个数**几乎占了全部概率**（接近1）
- 其他所有数**概率几乎为0**

这导致：**注意力变成"非0即1"的硬选择**，而不是"软"的注意力！模型就失去了"关注多个地方"的能力。

---

### ✅ 除以 √d 解决什么问题？

```
缩放后 = (Q · K) / √d
```

**数学原理**：

如果 Q 和 K 的每个维度都是**独立同分布**，均值为0，方差为1：

```
Var(Q · K) = d × 1 × 1 = d
标准差 = √d
```

除以 √d 后：

```
Var((Q · K) / √d) = d / d = 1
标准差 = 1
```

**现在数值回到正常范围**（均值0，方差1），Softmax 就不会"饱和"了！

---

### 📊 直观对比

| 情况 | 点积范围 | Softmax结果 | 问题 |
|------|---------|-------------|------|
| **不缩放** | [-50, 50] | 几乎 [0, 0, ..., 1, ..., 0] | 注意力太"硬"，只关注一个 |
| **除以√d** | [-3, 3] | [0.1, 0.2, 0.4, 0.2, 0.1] | 注意力"软"，合理分配 |

---

### 🎯 一句话总结

> **除以 √d 是为了让点积的方差保持为1，防止数值太大导致 Softmax 饱和，从而保持注意力的"软性"和多样性。**

## 🏗️ 第二部分：Transformer的Encoder和Decoder分别干啥的？

### 🎯 整体架构

Transformer 分为两大部分：

```
输入 → [Encoder] → 中间表示 → [Decoder] → 输出
```

**形象理解**：
- **Encoder（编码器）**：**阅读理解**。把输入（比如中文"你好"）读懂了，变成一个"语义向量"。
- **Decoder（解码器）**：**写作表达**。根据这个"语义向量"，一个字一个字地生成输出（比如英文"Hello"）。

---

### 📖 Encoder（编码器）详解

**输入**：完整的序列（比如一句话）

**目标**：把序列**理解**成一组"上下文感知"的向量

**结构**：

```
输入序列: [词1] [词2] [词3] ... [词n]
    ↓
┌──────────────────────────────────┐
│ 层1:                             │
│  ┌──────────────────────────┐    │
│  │  多头自注意力            │    │  ← 每个词看所有其他词（双向！）
│  │  (每个词看所有其他词)    │    │
│  └──────────────────────────┘    │
│           ↓                       │
│  ┌──────────────────────────┐    │
│  │  前馈网络 (FFN)          │    │  ← 一个普通的全连接层
│  │  进一步提炼信息          │    │
│  └──────────────────────────┘    │
└──────────────────────────────────┘
    ↓  （重复 N 次，比如6层）
    ↓
输出: [理解后的词1] [理解后的词2] ... [理解后的词n]
```

**关键特点**：
- **双向自注意力**：每个词都能看到**所有其他词**（左边和右边都看）
- **输出和输入长度相同**（都是 n 个向量）

---

### ✍️ Decoder（解码器）详解

**输入**：Encoder的输出 + 已经生成的前面几个词

**目标**：一个字一个字地**生成**输出序列

**结构**：

```
已经生成的词: [词1] [词2] [词3] ... [词t-1]
    ↓
┌──────────────────────────────────┐
│ 层1:                             │
│  ┌──────────────────────────┐    │
│  │  带掩码的多头自注意力    │    │  ← 每个词只能看"前面"的词
│  │  (只能看左边的词！)      │    │  ← 这就是"单向"的秘密！
│  └──────────────────────────┘    │
│           ↓                       │
│  ┌──────────────────────────┐    │
│  │  交叉注意力              │    │  ← 看Encoder的输出！
│  │  (和输入序列做注意力)    │    │  ← 这就是"翻译"的秘密！
│  └──────────────────────────┘    │
│           ↓                       │
│  ┌──────────────────────────┐    │
│  │  前馈网络 (FFN)          │    │
│  └──────────────────────────┘    │
└──────────────────────────────────┘
    ↓  （重复 N 次）
    ↓
输出: 下一个词 [词t] 的概率分布
```

---

### 🔑 Decoder 的两个关键注意力

#### 1️⃣ 带掩码的自注意力（Masked Self-Attention）

**问题**：训练时，Decoder知道"标准答案"（比如翻译任务里知道正确的英文句子）。如果不加限制，模型会"作弊"——直接看后面的词！

```
句子: "我 爱 你" → 翻译成 "I love you"

生成 "love" 时，不应该知道后面有 "you"！
否则就像考试偷看答案 ❌
```

**解决方案**：用**掩码（Mask）**挡住未来的词。

```
生成第3个词 "love" 时：
能看到: [I, love]  ← 只能看到已经生成的
看不到: [you]      ← 未来的词被掩码遮住了！
```

**矩阵形式**（注意力分数）：

```
         I    love   you
I       [1,   0.3,  0.1]   ← "I" 能看到自己、love、you？
love    [0.5, 0.8,  0.2]   ← "love" 能看到 I、自己、you？
you     [0.2, 0.6,  0.9]   ← "you" 能看到所有词？

↓ 加上掩码后 ↓

         I    love   you
I       [1,   0.3,  -∞]    ← "I" 只看自己
love    [0.5, 0.8,  -∞]    ← "love" 看 I 和自己
you     [0.2, 0.6,  0.9]   ← "you" 看所有（最后一个词）
```

**用 -∞ 代替分数**，Softmax 后就会变成 0，相当于"看不见"。

---

#### 2️⃣ 交叉注意力（Cross-Attention）

**作用**：Decoder 要**参考** Encoder 读到的输入信息。

**怎么做的**：
```
Decoder的Q ← 来自Decoder自己（当前要生成的词）
Encoder的K,V ← 来自Encoder的输出（整个输入序列的理解）
```

**形象理解**：
- 你写英文作文（Decoder），手里拿着中文原文（Encoder）
- 你要写下一个词时（Q），要回头查原文（K,V），看看对应什么
- 这就是"翻译"的本质！

---

### 📊 Encoder vs Decoder 对比

| 对比项 | Encoder | Decoder |
|--------|---------|---------|
| **目的** | 理解输入 | 生成输出 |
| **注意力** | 双向（看所有词） | 单向（只看左边）+ 交叉注意力 |
| **能看到谁** | 输入序列所有词 | 已生成的词 + Encoder的输出 |
| **输出** | 一组向量（和输入等长） | 一个词的概率分布 |
| **典型任务** | BERT（理解） | GPT（生成） |

## 💻 第三部分：实际代码实现（PyTorch版）

### 🧩 完整的自注意力代码

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class SelfAttention(nn.Module):
    def __init__(self, embed_dim, num_heads):
        """
        embed_dim: 词向量的维度（比如512）
        num_heads: 注意力头的数量（比如8）
        """
        super().__init__()
        assert embed_dim % num_heads == 0, "embed_dim 必须能被 num_heads 整除"
        
        self.embed_dim = embed_dim
        self.num_heads = num_heads
        self.head_dim = embed_dim // num_heads  # 每个头的维度（512/8=64）
        
        # Q, K, V 的线性变换层
        self.W_q = nn.Linear(embed_dim, embed_dim)
        self.W_k = nn.Linear(embed_dim, embed_dim)
        self.W_v = nn.Linear(embed_dim, embed_dim)
        
        # 输出投影层
        self.W_o = nn.Linear(embed_dim, embed_dim)
    
    def forward(self, x, mask=None):
        """
        x: [batch_size, seq_len, embed_dim]
        mask: [batch_size, seq_len, seq_len] 或 None
        """
        batch_size, seq_len, _ = x.size()
        
        # Step 1: 计算 Q, K, V
        Q = self.W_q(x)  # [batch, seq_len, embed_dim]
        K = self.W_k(x)  # [batch, seq_len, embed_dim]
        V = self.W_v(x)  # [batch, seq_len, embed_dim]
        
        # Step 2: 拆分成多头
        # 把 embed_dim 拆成 num_heads × head_dim
        Q = Q.view(batch_size, seq_len, self.num_heads, self.head_dim)
        K = K.view(batch_size, seq_len, self.num_heads, self.head_dim)
        V = V.view(batch_size, seq_len, self.num_heads, self.head_dim)
        
        # 调整维度顺序: [batch, num_heads, seq_len, head_dim]
        Q = Q.transpose(1, 2)
        K = K.transpose(1, 2)
        V = V.transpose(1, 2)
        
        # Step 3: 计算注意力分数
        # Q × K^T: [batch, num_heads, seq_len, head_dim] × [batch, num_heads, head_dim, seq_len]
        #         = [batch, num_heads, seq_len, seq_len]
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.head_dim ** 0.5)
        
        # Step 4: 应用掩码（如果需要）
        if mask is not None:
            # 把 mask 中为 0 的位置的分数设为 -inf
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        # Step 5: Softmax 归一化
        attention_weights = F.softmax(scores, dim=-1)  # [batch, num_heads, seq_len, seq_len]
        
        # Step 6: 加权求和
        output = torch.matmul(attention_weights, V)  # [batch, num_heads, seq_len, head_dim]
        
        # Step 7: 合并多头
        output = output.transpose(1, 2).contiguous()  # [batch, seq_len, num_heads, head_dim]
        output = output.view(batch_size, seq_len, self.embed_dim)  # [batch, seq_len, embed_dim]
        
        # Step 8: 输出投影
        output = self.W_o(output)  # [batch, seq_len, embed_dim]
        
        return output, attention_weights
```

### 🔧 完整的 Transformer 层（Encoder）

```python
class TransformerEncoderLayer(nn.Module):
    def __init__(self, embed_dim, num_heads, ff_dim, dropout=0.1):
        """
        embed_dim: 词向量维度
        num_heads: 注意力头数
        ff_dim: 前馈网络的隐藏层维度
        """
        super().__init__()
        
        # 多头自注意力
        self.self_attention = SelfAttention(embed_dim, num_heads)
        
        # 前馈网络（2层全连接）
        self.ffn = nn.Sequential(
            nn.Linear(embed_dim, ff_dim),
            nn.ReLU(),
            nn.Linear(ff_dim, embed_dim),
        )
        
        # 层归一化
        self.norm1 = nn.LayerNorm(embed_dim)
        self.norm2 = nn.LayerNorm(embed_dim)
        
        # Dropout（防止过拟合）
        self.dropout = nn.Dropout(dropout)
    
    def forward(self, x, mask=None):
        # 残差连接 + 层归一化
        # Step 1: 自注意力 + 残差
        attn_output, _ = self.self_attention(x, mask)
        x = self.norm1(x + self.dropout(attn_output))
        
        # Step 2: 前馈网络 + 残差
        ff_output = self.ffn(x)
        x = self.norm2(x + self.dropout(ff_output))
        
        return x
```

### 🚀 完整 Transformer（堆叠 N 层）

```python
class TransformerEncoder(nn.Module):
    def __init__(self, num_layers, embed_dim, num_heads, ff_dim, vocab_size, max_len):
        super().__init__()
        
        # 词嵌入
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        
        # 位置编码
        self.pos_encoding = self._create_positional_encoding(max_len, embed_dim)
        
        # 堆叠 N 层 Encoder
        self.layers = nn.ModuleList([
            TransformerEncoderLayer(embed_dim, num_heads, ff_dim)
            for _ in range(num_layers)
        ])
    
    def _create_positional_encoding(self, max_len, embed_dim):
        """生成位置编码"""
        pe = torch.zeros(max_len, embed_dim)
        position = torch.arange(0, max_len, dtype=torch.float).unsqueeze(1)
        div_term = torch.exp(torch.arange(0, embed_dim, 2).float() * 
                             (-math.log(10000.0) / embed_dim))
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        return pe.unsqueeze(0)  # [1, max_len, embed_dim]
    
    def forward(self, x):
        # x: [batch_size, seq_len]  (词索引)
        seq_len = x.size(1)
        
        # 词嵌入 + 位置编码
        x = self.embedding(x) + self.pos_encoding[:, :seq_len, :]
        
        # 通过 N 层 Encoder
        for layer in self.layers:
            x = layer(x)  # 双向注意力，没有mask
        
        return x
```

## 📖 第四部分：BERT 的原理（双向Transformer）

### 🎯 BERT 核心思想

> **"用海量文本做'完形填空'，让模型学会理解语言。"**

---

### 📊 预训练任务1：Masked Language Model（MLM）

**任务**：随机遮住15%的词，让模型猜。

```
原始句子: 我 爱 吃 苹 果

加 [MASK] 后: 我 爱 吃 [MASK] 果
              ↓
BERT 要预测: 苹   ✓
```

**具体怎么遮**：
- 80% 换成 [MASK]
- 10% 换成随机词
- 10% 保持不变

**为什么要这么复杂？**
- 全换 [MASK]：模型只在 [MASK] 处学，其他位置不学
- 部分随机：模型学会"即使有错词也要理解上下文"
- 部分保留：迫使模型"时刻准备着"预测任何词

---

### 📊 预训练任务2：Next Sentence Prediction（NSP）

**任务**：判断两句话是不是"上下句"。

```
句子A: 我昨天去了公园。
句子B: 那里有很多花。  → 是上下句 ✓
句子C: 我喜欢吃火锅。  → 不是上下句 ✗
```

**怎么做到的？**
- 输入：`[CLS] 句子A [SEP] 句子B [SEP]`
- 输出：`[CLS]` 位置的向量 → 分类（是/否）

---

### 🏗️ BERT 的网络结构

```
输入: [CLS] 我 爱 吃 [MASK] 果 [SEP]
        ↓
┌──────────────────────────────────┐
│ 词嵌入 + 位置嵌入 + 句子嵌入     │  ← 三层嵌入叠加
└──────────────────────────────────┘
        ↓
┌──────────────────────────────────┐
│ Transformer Encoder × 12层       │  ← 双向！每个词看所有词
│ (BERT-Base)                      │
└──────────────────────────────────┘
        ↓
┌──────────────────────────────────┐
│ [CLS]位置 → 分类输出            │  ← 用于 NSP 任务
│ [MASK]位置 → 预测被遮的词       │  ← 用于 MLM 任务
└──────────────────────────────────┘
```

---

### 🔑 BERT 的关键特点

| 特点 | 解释 |
|------|------|
| **双向** | 每个词都能看到左边和右边的词 |
| **只用了 Encoder** | 不需要 Decoder，因为不做生成 |
| **预训练+微调** | 先在海量数据上做"完形填空"，再在小任务上微调 |
| **词级别的理解** | 每个词都有对应的输出向量 |

---

### 🌰 BERT 的微调应用

| 任务 | 怎么用 BERT |
|------|------------|
| **情感分析** | 输入句子 → 取 `[CLS]` 输出 → 分类（好评/差评） |
| **问答系统** | 输入问题+文章 → BERT 预测"答案在文章哪个位置" |
| **命名实体识别** | 输入句子 → 每个词的输出 → 分类（人名/地名/组织名） |

---

## 💬 第五部分：GPT 的原理（单向Transformer）

### 🎯 GPT 核心思想

> **"用海量文本做'接龙'，让模型学会生成语言。"**

---

### 📊 预训练任务：语言建模（Language Modeling）

**任务**：给定前面的词，预测下一个词。

```
输入: 我 爱 吃
      ↓
GPT 预测: 苹 果  ✓
      ↓
继续: 我 爱 吃 苹 果
      ↓
GPT 预测: ，  ✓
      ↓
继续: 我 爱 吃 苹 果 ，
      ↓
GPT 预测: 很 甜  ✓
```

**这就是"自回归生成"**：一个字一个字地往外蹦！

---

### 🏗️ GPT 的网络结构

```
输入: 我 爱 吃 [MASK] 果
        ↓
┌──────────────────────────────────┐
│ 词嵌入 + 位置嵌入                │
└──────────────────────────────────┘
        ↓
┌──────────────────────────────────┐
│ Transformer Decoder × 12层       │  ← 单向！只能看左边
│ (带掩码的自注意力)              │
└──────────────────────────────────┘
        ↓
┌──────────────────────────────────┐
│ 最后一个词的输出 → 预测下一个词  │
│ 概率分布：[苹:0.8, 香:0.1, ...] │
└──────────────────────────────────┘
```

---

### 🔑 GPT 的关键特点

| 特点 | 解释 |
|------|------|
| **单向** | 每个词只能看到左边的词（因为生成时不知道未来） |
| **只用了 Decoder** | 不需要 Encoder，因为不需要"理解"输入 |
| **生成式** | 目标是"生成"而不是"理解" |
| **自回归** | 一个个词往外吐，每次都用之前生成的词 |

---

### ⚡ GPT 的"零样本学习"能力

**问题**：BERT 每个任务都要微调，太麻烦！

**GPT 的解决方案**：用"提示词（Prompt）"告诉它要干什么。

```
翻译任务：
输入: "Translate to English: 你好"
GPT: "Hello"  ✓

摘要任务：
输入: "Summarize: 今天天气很好，我们去公园玩..."
GPT: "今日游玩愉快"  ✓

代码生成：
输入: "Write a Python function to sort a list"
GPT: "def sort_list(lst): return sorted(lst)"  ✓
```

**同一个模型，不同的任务，不需要重新训练！**

### 🎯 GPT 的 RLHF（人类反馈强化学习）

**问题**：GPT-3 生成的内容"很聪明"，但可能**不友善、不准确、不安全**。

**RLHF 的解决步骤**：

```
Step 1: 用人类偏好训练一个"奖励模型"
        人类给 GPT 的多个回答打分
        奖励模型学会"模仿人类的判断"

Step 2: 用强化学习微调 GPT
        GPT 生成回答 → 奖励模型打分 → 高分加强，低分减弱
        就像训练小狗：做对了给零食，做错了不给
```

**结果**：ChatGPT 不仅聪明，还**友善、诚实、安全**！

## ⚖️ BERT vs GPT 深度对比

| 对比项 | BERT | GPT |
|--------|------|-----|

| **擅长任务** | 理解、分类、问答 | 生成、聊天、创作 |
| **任务适应方式** | 微调（Fine-tuning） | 提示词（Prompting） |
| **是否能生成** | 不能（只能理解） | 能（生成超强） |

## ✅ 终极总结

### Transformer 全家桶

1. **Transformer = 地基** → 自注意力 + 多头注意力 + 位置编码
2. **BERT = 理解专家** → Encoder 堆叠，双向，做分类/问答
3. **GPT = 生成专家** → Decoder 堆叠，单向，做聊天/写作

### 数学核心

- **除以 √d**：防止 Softmax 饱和，保持注意力"软性"
- **残差连接**：帮助梯度传播，防止过平滑
- **层归一化**：稳定训练，加速收敛

### 为什么这么厉害？

- **并行计算**：比 RNN 快 10 倍以上
- **长距离依赖**：每个词直接看所有词
- **可扩展性**：越大越强（GPT-3 1750亿参数）
- **迁移学习**：预训练+微调/提示词，一个模型干所有事
