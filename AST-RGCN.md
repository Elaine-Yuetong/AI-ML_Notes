
## 🌳 第一部分：AST（抽象语法树）

### 什么是AST？

**AST就是把一个数学公式画成一棵树**，让你一眼看清它的"骨架结构"。

### 举个栗子 🌰

假设有个公式：
```
(Close - Open) / Open
```
意思是：（收盘价 - 开盘价）÷ 开盘价

#### 作为"一串文字"看（不好）：
```
"Close - Open / Open"
```
这样看会搞混，因为不知道先算减法还是除法。

#### 作为"树"看（好）：
```
        ÷
       / \
      -   Open
     / \
  Close Open
```
这棵树清清楚楚地告诉我们：
1. 先算 `Close - Open`
2. 再把结果除以 `Open`

---

### 为什么要用AST？（重要！）

| 方式 | 问题 | AST怎么解决 |
|------|------|------------|
| **序列（一串字）** | `(a+b)` 和 `(b+a)` 看起来不一样 | 树结构一样，模型能看懂"它们是一回事" |
| **序列** | 太长的公式容易"记不住前面" | 树是分层的，每层关系清晰 |
| **序列** | 看不出"哪个操作先算" | 树天生就表达了计算顺序 |

---

### 再举个复杂点的栗子 🌰

公式：`TsMean(Volume, 5) + Close`

意思是：过去5天的平均成交量 + 收盘价

它的AST长这样：
```
        +
       / \
      /   \
  TsMean  Close
   /   \
Volume  5
```

你看，树把"操作符"（+、TsMean）和"数据"（Volume、Close、5）清晰地分开了。

## 🧠 第二部分：RGCN（关系图卷积网络）

### 先理解GCN（图卷积网络）

GCN就是**一种能"看懂"图（或树）的神经网络**。

**核心思想**：每个节点（比如树里的 `+`、`Close`）会**听邻居的话**，然后更新自己的"理解"。

#### 举个栗子 🌰

想象一个班级：
- 每个学生是一个**节点**
- 朋友关系是**边**
- GCN做的就是：每个学生**听听朋友们说了啥**，然后综合一下，形成自己的新看法

经过好几轮"互相听"之后，每个学生都**知道了整个班级的情况**。

---

### 那RGCN和GCN有什么不同？

**关键区别**：RGCN会区分"不同类型的关系"。

#### 还是举栗子 🌰

在班级里：
- **好朋友**：说的话分量重
- **普通同学**：说的话分量轻
- **老师**：说的话分量更重

GCN：所有人说的话**一样重要** ❌

RGCN：**不同关系，不同待遇** ✅

---

### 在Alpha公式里，RGCN怎么用？

回到AST，我们有不同类型的"关系"：

```
        ÷
       / \
      -   Open
     / \
  Close Open
```

RGCN会区分：

| 关系类型 | 例子 | 解释 |
|---------|------|------|
| **一元操作** | `Abs(Close)` | 一个操作符对一个数 |
| **二元操作-左** | `Close - Open` 中的 `Close` | 减法左边 |
| **二元操作-右** | `Close - Open` 中的 `Open` | 减法右边 |
| **滚动操作-数据** | `TsMean(Volume, 5)` 中的 `Volume` | 时间序列操作的数据部分 |
| **滚动操作-时间** | `TsMean(Volume, 5)` 中的 `5` | 时间序列操作的时间参数 |

---

### RGCN的工作流程（超简化版）

**第1步**：每个节点（`+`、`-`、`Close`等）有一个**初始向量**（相当于一个"身份证号"）。

**第2步**：每个节点**问邻居**："你是谁？咱们是什么关系？"

**第3步**：根据**关系类型**，用**不同的"翻译器"**（权重矩阵）处理邻居的信息。

**第4步**：把所有邻居的信息**加起来**，再加上自己的信息，形成**新的向量**。

**第5步**：重复2-4步好几轮（论文里用了2层）。

**第6步**：最后把所有节点的向量**压缩成一个**（用MaxPooling），这就是**整个公式的"指纹"**。

---

### 为什么要区分关系类型？

看这两个公式：

**公式A**：`Close - Open`
```
    -
   / \
Close Open
```

**公式B**：`Open - Close`
```
    -
   / \
Open Close
```

如果不区分左右关系，RGCN会认为A和B**一模一样**（因为节点和邻居都一样）。

但实际上它们**完全不同**！一个表示"收盘-开盘"，一个表示"开盘-收盘"。

RGCN区分了"左操作数"和"右操作数"，所以能**正确区分**这两个公式。

---

## 🔗 AST + RGCN 组合起来干嘛？

```
公式 → 画成AST（树） → RGCN读懂这棵树 → 输出一个"指纹向量"
```

这个"指纹向量"有什么用？

1. **判断两个公式像不像**：指纹距离近 = 结构相似
2. **指导生成**：生成新公式时，看看这个指纹好不好
3. **奖励计算**：如果指纹和行为对得上，给加分

> **AST是把公式画成一棵树，让计算机看清楚"骨架"；RGCN是能读懂这棵树的神经网络，还特别聪明地知道"不同的树枝关系"要区别对待。两者合体，模型就真正理解了公式的"意思"，而不是死记硬背一串字符。**
